Course Description &
Understanding the concepts and theory of database systems. Topics include
CLOs
database system architectures, data models, query languages, conceptual and
logical design, physical organization, transaction management, and query
processing. The entity-relationship model and relational model are discussed
in detail for the basis of developing the conceptual model. The mapping from
the conceptual level to the logical level, integrity constraints, functional
dependencies and normalization are studied as a basis for formal design.
Theoretical languages such as the relational algebra and the relational calculus
are described, and high-level language, SQL, is discussed. File organization
and access methods are provided as a basis for discussion of query
optimization techniques. Transaction processing techniques are presented with a
specific emphasis on concurrency control. Emerging approaches & technologies
such as NoSQL, Data Warehousing & Business Intelligence, are also discussed.
The course learning objectives are preparing students for effective use of
database technology for data management with hands-on-experience in a
team work environment.
After completion of the course, the student shall be able to:
1. Learn fundamental concepts, architecture and components of database
systems.
2. Evaluate relational model, relational algebra and constraints on data with
real life examples.
3. To design a conceptual models using ER/ EER Diagram for a real life
database applications.
4. Implement database schema of ER/EER Diagrams.
5. Test conceptual models or design using functional dependency and
normalization.
6. Learn the concepts of database programming, SQL.
7. Learn fundamental concepts of transaction processing & concurrency.
8. Learn fundamental concepts of query processing & optimization.
9. Learn emerging approaches & technologies such as NoSQL, Big Data, Data
Warehousing & Business Intelligence, Distributed Data Management.
